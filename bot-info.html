Telegram Notification Bot - Overview

Any Telegram user can request /myid - the bot returns their Telegram ID. With this ID, the user contacts an administrator (any authorized user) to be added to the whitelist. After being added, the user becomes authorized and can use the botâ€™s functionality.

Authorization Logic

  • If the whitelist is not empty, the bot responds only to users in the whitelist.
  • /myid is always available to everyone.
  • If the whitelist is empty, the bot responds to everyone.

Core Behavior

  • The bot accepts /start.
  • Interactive commands work via state machine (step-by-step flow).
  • Unknown commands outside active state trigger a hint to use /start.

Features

  • /links - display stored links
  • /chats - list known chats (auto-saved on any message)
  • /templates - manage templates(support variables) (add / edit / delete)
  • /users - manage whitelist (add / list / delete)
  • /myid - return Telegram ID and username
  • /sendNotification - step-by-step flow:
    • Select template
    • Select chat(s)
    • Enter variables
    • Preview
    • Confirm sending (yes/no)

Deployment

  • The bot is currently hosted locally.
  • The computer/server running it must remain powered on and connected to the internet.
  • For continuous availability, it can be deployed to a VPS or cloud production server.
